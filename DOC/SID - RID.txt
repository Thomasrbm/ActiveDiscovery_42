


SID = Security Identifier 


c est le vrai ID que windows use. pour permisions etc etc.
les noms sont des alias.




S-1-5-21-3623811015-3361044348-30300820-1013
│ │ │  │                                │
│ │ │  └─ Domain Identifier             └─ RID
│ │ └─ Authority (5 = NT Authority)
│ └─ Revision
└─ Préfixe SID


3 grand nombres = domain identifier






* Revision :  numero de la version pour le forma SID
pas encore changed, maybe un jour





* RID = Relative IDentifier





* Authority :  dit qui gere ce SID

┌─────────┬──────────────────────┬────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│ AUTH N° │ NOM DE L'AUTHORITY   │ À QUOI ÇA SERT                                                                                                     │
├─────────┼──────────────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│   0     │ Null Authority       │ SID "vide", quasi jamais utilisé                                                                                   │
│   1     │ World Authority      │ Identités universelles ("tout le monde a le droit")                                                                │
│   2     │ Local Authority      │ seul les gens connecter physiquement sur CE pc auront acces au fichier                                             │
│   3     │ Creator Authority    │ createur aura droit speciaux, mais sous droits pour autres users                                                   │
│   4     │ Non-unique Authority │ Obsolète, ignore                                                                                                   │
│   5     │ NT Authority         │ ⭐ LA PLUS COMMUNE — ce sid est active directory                                                                   │
│   9     │ Resource Manager     │ les daemon de windows only                                                                                         │
│  15     │ App Package          │ Ce user a launch app Microsoft Store. Cette app n'a le droit  qu'à ses propres fichiers                            │
│  16     │ Mandatory Label      │ Niveaux d'intégrité/habilitation UAC (Low/Med/High/Sys)                                                            │
│  18     │ Asserted Identity    │ Tokens Windows modernes (biometrique windowsHello, etc.), ajoute un token de plus que mdp potentiellement voled    │
└─────────┴──────────────────────┴────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘


niveau UAC : (comme cercle privilege linux)

Attribued par le windows kernel 


LOW : S-1-16-4096   =  navigateur web (Edge, Chrome) ou lecteurs de PDF   => read only et ecriture dossier temps
Medium : S-1-16-8192   =  niveau normal quand ouvre session   =>  RWX mais que propres fichiers
High : S-1-16-12288   =  powershell exec en tant qu admin   => passe droit partout SAUF windows kernel
System : S-1-16-16384   =  niveau exclusif des daemons système  => dieu
